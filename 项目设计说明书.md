# 房产管理系统 项目设计说明书

## 1. 项目概述

### 1.1 项目名称

房产管理系统 (Estate Management System)

### 1.2 项目背景

本项目为数据库课程设计作业，旨在开发一套完整的房产管理系统，实现高校教工住房的分房、调房、退房及咨询统计功能。系统采用 B/S 架构，后端使用 Spring Boot + MyBatis-Plus 框架，前端使用 Vue.js 框架，数据库采用 MySQL。

### 1.3 系统目标

- 实现分房申请的录入、评分、排队和自动分配功能
- 实现退房申请的处理和房屋回收功能
- 实现调房申请的置换处理功能
- 提供住房信息查询和统计报表功能
- 支持房产科对系统参数的维护（如房租调整、标准修改）
- **加分项**：集成 Redis 缓存、分布式锁、多线程异步处理、敏感数据加解密

### 1.4 适用范围

本系统适用于高校房产管理部门对教工住房进行日常管理和月度分配活动。

## 2. 需求分析

### 2.1 功能性需求

| 编号 | 功能模块 | 功能描述 |
|------|---------|---------|
| F1 | 申请提交 | 住户提交分房/调房/退房申请，系统检查合法性并校验分数阈值 |
| F2 | 分房处理 | 每月最后一天自动分房，按分数高低从分房队列匹配空房 |
| F3 | 退房处理 | 处理退房申请，清理住户信息和账单，回收房屋 |
| F4 | 调房处理 | 处理调房申请，退还原住房，分配新住房 |
| F5 | 信息查询 | 住户查询阈值分数、居住条件、房租等信息 |
| F6 | 统计报表 | 房产科打印住房情况统计表 |
| F7 | 参数管理 | 房产科修改居住条件、单位面积房租等参数 |

### 2.2 非功能性需求

- 响应时间：查询操作 <= 2 秒
- 数据一致性：分房活动使用事务和分布式锁确保数据一致
- 可用性：系统支持 7x24 小时运行
- 安全性：敏感数据 AES 加密存储
- 并发性：月底分房活动支持多线程并行处理

## 3. 系统架构设计

### 3.1 总体架构

采用分层架构（Layered Architecture），自顶向下分为四层：

- **前端层**：Vue.js (Element UI / Axios / Vue Router)
- **Controller 层**：RESTful API 接口层
- **Service 层**：业务逻辑层 + @Transactional 事务
- **Mapper 层**：MyBatis-Plus 数据访问层
- **数据库层**：MySQL

## 4. 业务流程设计

### 4.1 分房流程

```
提交分房申请 -> 校验合法性 -> 计算分数(职称+人口) -> 阈值比较
-> 插入队列(按分数排序) -> 每月最后一天自动执行:
   1. 获取分布式锁防止并发
   2. 从空房表读取空房
   3. 按队列顺序匹配最佳房源
   4. 更新房产状态、创建住户、生成账单
   5. 输出住房分配单
```

### 4.2 退房流程

```
提交退房申请 -> 校验住户信息匹配 -> 删除住户记录
-> 删除房租账单 -> 房屋置为空闲 -> 更新申请单状态
```

### 4.3 调房流程

```
提交调房申请 -> 校验原住房信息 -> 退掉原住房(置为空闲)
-> 按分房算法查找目标空房 -> 分配新房 -> 更新住户信息、生成新账单
```

## 5. 核心算法设计

### 5.1 住房分数计算

```
基础分(按职称): 教授/处长=100, 科长/研究员=80,
               讲师/工程师=60, 实验员/干事=50, 其他=40
住房分数 = 基础分 + 家庭人口 x 10
```

### 5.2 智能找房算法

1. 查询所有空房且面积 >= 申请面积，按面积降序、租金升序排列
2. 加载住房标准表（面积->最低分数映射）
3. 遍历候选房源，返回第一个满足分数阈值的房源
4. 无匹配则返回 null（保留在队列中等待）

## 6. 加分项实现说明

### 6.1 事务管理
所有写操作（分房/退房/调房/批量分房）均使用 `@Transactional` 注解，确保多表操作的原子性。

### 6.2 Redis 缓存
查询类操作使用 `@Cacheable` 注解缓存结果，写操作使用 `@CacheEvict` 清除缓存。默认 TTL 30 分钟。

### 6.3 分布式锁
月底分房使用 Redisson 分布式锁（lock:monthlyAllocation），确保多实例互斥执行。

### 6.4 多线程
配置线程池（@Async + @EnableAsync），月底分房支持异步并行处理。

### 6.5 敏感数据加解密
使用 Hutool AES 工具类对住户姓名、部门等敏感信息进行加解密。

## 8. 部署说明

```bash
# 方式一：手动初始化数据库
mysql -u root -p < 数据库/建表.sql
mysql -u root -p < 数据库/初始化数据.sql

# 方式二：Spring Boot 自动初始化（需启动 Redis）
cd 后端服务代码/Demo
mvn spring-boot:run
# 启动时自动执行 schema.sql + data.sql

# 前端启动
cd 前端代码
npm install && npm run serve
```
