# 房产管理系统 — 数据库设计报告

> **数据库**：estate_management  
> **DBMS**：MySQL 8.0 (InnoDB, utf8mb4)  
> **表数量**：5 张  
> **范式**：第三范式 (3NF)

---

## 一、概念结构设计 (E-R 模型)

### 1.1 实体列表

| 实体 | 描述 | 主键 |
|------|------|------|
| 住房标准 (HouseStandard) | 住房面积对应的最低分数阈值 | `id` (自增)，`area` 唯一 |
| 房产 (House) | 房屋基本信息 | `house_no` |
| 住户 (Resident) | 当前在住人员信息 | `id` (自增)，`house_no` 唯一 |
| 房租账单 (RentBill) | 每月房租费用记录 | `id` (自增) |
| 申请单 (Application) | 分房/调房/退房申请 | `apply_no` |

### 1.2 E-R 关系图

```
                         ┌──────────────────┐
                         │  house_standard   │
                         │  面积 / 最低分数   │
                         └────────┬─────────┘
                                  │ (面积参考)
                                  ▼
┌──────────────┐  1:1   ┌──────────────┐  1:N   ┌──────────────┐
│   resident   │◄───────│    house     │        │  rent_bill   │
│  户主/部门    │        │  房号/面积    │        │ 月份/金额     │
│  职称/分数    │        │  状态/租金    │        └──────────────┘
└──────┬───────┘        └──────┬───────┘
       │ (户主姓名)             │ (房号)
       │                       │
       └───────────┬───────────┘
                   │
                   ▼
         ┌──────────────────┐
         │   application    │
         │  申请类型/状态    │
         │  分数/面积/房号   │
         │  驳回原因         │
         └──────────────────┘
```

### 1.3 实体关系说明

| 关系 | 类型 | 说明 |
|------|:--:|------|
| house → resident | **1:1** | 一套房最多住一户（`house_no` UNIQUE） |
| house → rent_bill | **1:N** | 一套房有多月账单 |
| resident → application | **1:N** | 一个住户可多次申请 |
| house → application | **N:1** | 申请关联原房号或分配房号 |
| house_standard → house | **1:N** | 一个面积标准对应多套该面积的房 |

---

## 二、逻辑结构设计（关系模式）

所有关系均满足 **3NF**（主键单属性，无部分依赖，无传递依赖）。

```
1. 住房标准 (id, area, min_score)
   PK: id   UK: area

2. 房产 (house_no, area, status, rent_per_sqm)
   PK: house_no

3. 住户 (id, owner_name, department, title, family_size, score, house_no, house_area)
   PK: id   UK: house_no   FK: house_no → house

4. 房租账单 (id, house_no, owner_name, bill_month, rent_amount, create_time)
   PK: id   FK: house_no → house

5. 申请单 (apply_no, applicant_name, department, apply_type, apply_status,
           req_area, old_house_no, old_house_area, allocated_house_no,
           title, family_size, score, reject_reason, create_time, process_time)
   PK: apply_no
```

## 三、物理表结构

### 3.1 住房标准表 `house_standard`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK AUTO_INCREMENT | 主键 |
| area | DECIMAL(10,2) | NOT NULL, UNIQUE | 住房面积 (㎡) |
| min_score | INT | NOT NULL | 该面积对应的最低分数 |

```sql
CREATE TABLE house_standard (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    area DECIMAL(10,2) NOT NULL UNIQUE COMMENT '住房面积（㎡）',
    min_score INT NOT NULL COMMENT '最低住房分数阈值'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='住房标准表';
```

### 3.2 房产表 `house`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| house_no | VARCHAR(50) | PK | 房号 |
| area | DECIMAL(10,2) | NOT NULL | 面积 (㎡) |
| status | INT | NOT NULL | 0=空闲 / 1=已分配 |
| rent_per_sqm | DECIMAL(10,2) | NOT NULL | 每平米月租金 (元) |

```sql
CREATE TABLE house (
    house_no VARCHAR(50) PRIMARY KEY COMMENT '房号',
    area DECIMAL(10,2) NOT NULL COMMENT '住房面积（㎡）',
    status INT NOT NULL COMMENT '分配标志：0-空闲, 1-已分配',
    rent_per_sqm DECIMAL(10,2) NOT NULL COMMENT '每平米房租（元/㎡）'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='房产表';
```

### 3.3 住户表 `resident`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK AUTO_INCREMENT | 主键 |
| owner_name | VARCHAR(50) | NOT NULL | 户主姓名 (**AES加密存储**) |
| department | VARCHAR(100) | NOT NULL | 所属部门 (**AES加密存储**) |
| title | VARCHAR(50) | NOT NULL | 职称 |
| family_size | INT | NOT NULL | 家庭人口 |
| score | INT | NOT NULL | 住房分数 |
| house_no | VARCHAR(50) | NOT NULL, UNIQUE, FK | 房号（一户一房） |
| house_area | DECIMAL(10,2) | NOT NULL | 住房面积 |

```sql
CREATE TABLE resident (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    owner_name VARCHAR(50) NOT NULL COMMENT '户主姓名（AES加密）',
    department VARCHAR(100) NOT NULL COMMENT '所属部门（AES加密）',
    title VARCHAR(50) NOT NULL COMMENT '职称',
    family_size INT NOT NULL COMMENT '家庭人口',
    score INT NOT NULL COMMENT '住房分数',
    house_no VARCHAR(50) NOT NULL UNIQUE COMMENT '房号',
    house_area DECIMAL(10,2) NOT NULL COMMENT '住房面积',
    FOREIGN KEY (house_no) REFERENCES house(house_no)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='住户表';
```

> ⚠️ `owner_name` 和 `department` 使用 AES-128 对称加密存储，应用层自动加解密，数据库层面看到的是密文。

### 3.4 房租表 `rent_bill`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK AUTO_INCREMENT | 账单ID |
| house_no | VARCHAR(50) | NOT NULL, FK | 房号 |
| owner_name | VARCHAR(50) | NOT NULL | 户主姓名 |
| bill_month | VARCHAR(10) | NOT NULL | 账单月份 (yyyy-MM) |
| rent_amount | DECIMAL(10,2) | NOT NULL | 房租金额（面积×单价） |
| create_time | DATE | — | 创建日期 |

```sql
CREATE TABLE rent_bill (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    house_no VARCHAR(50) NOT NULL COMMENT '房号',
    owner_name VARCHAR(50) NOT NULL COMMENT '户主姓名',
    bill_month VARCHAR(10) NOT NULL COMMENT '账单月份',
    rent_amount DECIMAL(10,2) NOT NULL COMMENT '房租金额（元）',
    create_time DATE COMMENT '创建时间',
    FOREIGN KEY (house_no) REFERENCES house(house_no)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='房租表';
```

### 3.5 申请表 `application`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| apply_no | VARCHAR(50) | PK | 申请编号 (APP-时间戳) |
| applicant_name | VARCHAR(50) | NOT NULL | 申请人姓名 |
| department | VARCHAR(100) | NOT NULL | 部门 |
| apply_type | INT | NOT NULL | 1=分房 / 2=调房 / 3=退房 |
| apply_status | INT | NOT NULL | 0=待处理 / 1=已通过 / 2=已驳回 |
| req_area | DECIMAL(10,2) | — | 要求面积（分房/调房） |
| old_house_no | VARCHAR(50) | — | 原房号（调房/退房） |
| old_house_area | DECIMAL(10,2) | — | 原面积（调房） |
| allocated_house_no | VARCHAR(50) | — | 分配房号 |
| title | VARCHAR(50) | — | 职称（分房/调房） |
| family_size | INT | — | 家庭人口（分房/调房） |
| score | INT | — | 住房分数（系统计算） |
| reject_reason | VARCHAR(500) | — | 驳回原因 |
| create_time | DATETIME | — | 创建时间 |
| process_time | DATETIME | — | 处理时间 |

```sql
CREATE TABLE application (
    apply_no VARCHAR(50) PRIMARY KEY COMMENT '申请单编号',
    applicant_name VARCHAR(50) NOT NULL COMMENT '申请人姓名',
    department VARCHAR(100) NOT NULL COMMENT '部门',
    apply_type INT NOT NULL COMMENT '申请类型：1-分房 2-调房 3-退房',
    apply_status INT NOT NULL COMMENT '处理状态：0-待处理 1-已通过 2-已驳回',
    req_area DECIMAL(10,2) COMMENT '要求住房面积',
    old_house_no VARCHAR(50) COMMENT '原房号',
    old_house_area DECIMAL(10,2) COMMENT '原住房面积',
    allocated_house_no VARCHAR(50) COMMENT '分配房号',
    title VARCHAR(50) COMMENT '职称',
    family_size INT COMMENT '家庭人口',
    score INT COMMENT '住房分数',
    reject_reason VARCHAR(500) COMMENT '驳回原因',
    create_time DATETIME COMMENT '创建时间',
    process_time DATETIME COMMENT '处理时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='申请表';
```

## 五、约束与完整性

| 类型 | 约束 | 实现 |
|------|------|------|
| 实体完整性 | 每表设 PRIMARY KEY | DDL |
| 参照完整性 | resident.house_no → house.house_no | FOREIGN KEY |
| 参照完整性 | rent_bill.house_no → house.house_no | FOREIGN KEY |
| 唯一约束 | house_no 在 resident 表中 UNIQUE（一户一房） | DDL |
| 业务约束 | apply_type ∈ {1,2,3} | 程序校验 |
| 业务约束 | apply_status ∈ {0,1,2} | 程序校验 |
| 业务约束 | house.status ∈ {0,1} | 程序校验 |
| 业务约束 | 分房分数 ≥ 对应面积阈值 | ApplicationService 提交时校验 |
| 业务约束 | 退房/调房须验证住户存在+房号匹配 | ApplicationService 审批时校验 |
| 数据安全 | owner_name / department AES 加密 | AesUtil 自动加解密 |

## 六、索引策略

| 表 | 索引 | 类型 | 用途 |
|------|------|------|------|
| house | PRIMARY (house_no) | 聚簇 | 按房号查询 |
| house | status | 隐式 | `WHERE status = 0` 空房查询 |
| resident | UNIQUE (house_no) | 唯一 | 按房号查住户 |
| rent_bill | bill_month | 隐式 | 按月查询账单 |
| application | (apply_type, apply_status) | 复合 | 分房队列查询 `WHERE type=1 AND status=0 ORDER BY score DESC` |
| application | create_time | 隐式 | 按时间排序 |
