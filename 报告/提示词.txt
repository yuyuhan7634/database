# 一、核心修改原则（写法三）

1. **每张表对应一个 Mapper**：

   * `ApplicationMapper` → `application` 表
   * `HouseMapper` → `house` 表
   * `HouseStandardMapper` → `house_standard` 表
   * `ResidentMapper` → `resident` 表
   * `RentBillMapper` → `rent_bill` 表

2. **不使用 XML，也不使用 `@Select/@Insert` 注解**：

   * 使用 MyBatis-Plus 提供的 `BaseMapper` 自带方法：

     * `insert()`
     * `updateById()`
     * `selectById()`
     * `selectList(QueryWrapper)`
   * 复杂查询使用 `QueryWrapper` 构建条件。

3. **Service 层处理业务逻辑**：

   * 分房/调房/退房逻辑都在 Service 层。
   * 保证**事务**：

     ```java
     @Transactional(rollbackFor = Exception.class)
     public void allocateHouse(Application app) { ... }
     ```

4. **Controller 层只负责接收请求、调用 Service、返回结果**。

# 二、具体修改建议

## 1. ApplicationService

### 问题与修改建议：

| 功能     | 建议                                                                                                                |
| ------ | ----------------------------------------------------------------------------------------------------------------- |
| 分房     | 使用 `HouseMapper.selectList(QueryWrapper)` 获取空房列表；按面积 DESC、房号 ASC 排序；循环匹配申请分数和最低分数；直接通过 `insert` 和 `updateById` 修改 |
| 调房     | 先从 `ResidentMapper` 查原房信息，验证面积；再通过 `HouseMapper` 找新房；更新原房状态；插入新住户和账单                                              |
| 退房     | 从 `ResidentMapper` 删除住户信息；从 `RentBillMapper` 删除账单（按姓名+房号）；更新 `HouseMapper` 空房状态                                   |
| 分房队列   | 使用 `ApplicationMapper.selectList(QueryWrapper)` 按分数 DESC 排序，无需 XML                                                |
| 驳回申请   | 计算分数低于最低分数时，直接抛异常，不回滚已保存的记录，或者按业务保存为 `status=2`                                                                   |
| 月底自动分房 | 用 `@Scheduled` 调用分房方法；可用 QueryWrapper 获取待分房申请列表                                                                   |

示例：

```java
QueryWrapper<House> houseQuery = new QueryWrapper<>();
houseQuery.eq("status", 0)
          .ge("area", reqArea)
          .orderByDesc("area")
          .orderByAsc("house_no");
List<House> availableHouses = houseMapper.selectList(houseQuery);
```

## 2. HouseService

* 移除 `HouseStandardMapper` 注入。
* 使用 `HouseMapper` 提供的 `QueryWrapper` 查询和修改 `house_standard`。
* 示例：

```java
QueryWrapper<HouseStandard> wrapper = new QueryWrapper<>();
wrapper.orderByAsc("area");
List<HouseStandard> standards = houseMapper.selectList(wrapper);
```

* 插入/更新标准：

```java
houseMapper.insert(standard);
houseMapper.updateById(standard);
```

---

## 3. ResidentService

* 移除 `RentBillMapper` 注入。
* 使用 `ResidentMapper` 来查询住户和账单。
* 查询示例：

```java
QueryWrapper<Resident> query = new QueryWrapper<>();
query.eq("owner_name", ownerName).eq("house_no", houseNo);
Resident resident = residentMapper.selectOne(query);
```

* 删除账单：

```java
QueryWrapper<RentBill> billQuery = new QueryWrapper<>();
billQuery.eq("owner_name", ownerName).eq("house_no", houseNo);
rentBillMapper.delete(billQuery);
```

---

## 4. Mapper 层

* 保持每张表独立 Mapper。
* Mapper 方法仅使用 MyBatis-Plus 内置方法。
* **删除所有 XML 文件**，不依赖注解 SQL。

---

## 5. Controller 层

* 保持原分层。
* Controller 只调用 Service。
* 返回统一 `Result<T>`。
* 示例：

```java
@PostMapping("/application/submit")
public Result<String> submitApplication(@RequestBody ApplicationSubmitDTO dto) {
    applicationService.submitApplication(dto);
    return Result.success("申请提交成功");
}
```

---

## 6. 数据库操作

* 全部通过 Mapper 的 `BaseMapper` + `QueryWrapper` 实现。
* 不用 XML 或注解 SQL。
* 复杂逻辑如“按分数排序队列”通过 `QueryWrapper.orderByDesc("score").orderByAsc("create_time")` 实现。
* 事务通过 `@Transactional` 管理。

---

## 7. 注意点

1. **分房队列**：查询时排序即可，不需要额外队列表。
2. **原房面积校验**：调房申请必须验证原房面积。
3. **退房账单删除**：一定要加 `house_no`，避免误删。
4. **分页或排序**：查询列表可用 `QueryWrapper.orderByAsc()`。
5. **敏感信息加密**：可用 `EncryptUtil` 加密字段，查询时解密。
6. **加分项**：

   * Redis 缓存空房列表或分房队列
   * Redisson 分布式锁防止重复分配
   * 多线程可用于批量分房或统计报表

✅ **结论**

* 保留现有 Mapper / Service / Controller / Entity 分层，不合并。
* XML 可以全部删除，使用 MyBatis-Plus 自带方法 + QueryWrapper 完成所有 SQL。
* Service 层逻辑清楚，Controller 只调用 Service。
* 前端 Vue 可以直接调用 Controller 提供的 REST API。
* 这是“写法三”，简洁且符合课程设计逻辑，便于答辩和扩展加分项。
