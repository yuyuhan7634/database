# 房产管理系统 数据库设计报告

## 1. 数据库概述

- 数据库名称：estate_management
- DBMS：MySQL 8.0+（InnoDB, utf8mb4）
- 设计原则：满足第三范式(3NF)，消除数据冗余，保证完整性和一致性

---

## 2. 概念结构设计（E-R 模型）

### 实体列表

| 实体 | 描述 | 标识符 |
|------|------|--------|
| 住房标准(HouseStandard) | 面积对应的最低分数阈值 | area |
| 房产(House) | 房屋信息（面积、状态、租金） | house_no |
| 住户(Resident) | 居住人员信息 | owner_name |
| 房租账单(RentBill) | 每月房租费用 | id |
| 申请单(Application) | 分房/调房/退房申请 | apply_no |

### 实体间关系

```
住房标准 --1:N-- 房产（一种面积标准对应多套房屋）
房产     --1:1-- 住户（一套房屋最多一户居住）
住户     --1:N-- 房租账单（一户有多月账单）
申请单   --N:1-- 房产（可选，关联原房屋）
申请单   --N:1-- 住房标准（可选，关联目标面积）
```

---

## 3. 逻辑结构设计

### 3.1 关系模式

1. 住房标准(**面积**, 最低分数)
2. 房产(**房号**, 面积, 分配标志, 每平方米房租)
3. 住户(**户主姓名**, 部门, 职称, 家庭人口, 住房分数, 房号, 住房面积)
4. 房租(**账单ID**, 房号, 户主姓名, 账单月份, 房租金额, 创建时间)
5. 申请单(**申请单号**, 申请人姓名, 部门, 申请类型, 处理状态, 要求面积, 原房号, 原住房面积, 职称, 家庭人口, 住房分数, 驳回原因, 创建时间, 处理时间)

### 3.2 函数依赖与范式

所有关系模式均满足 **3NF**：
- 主键均为单属性，无部分函数依赖
- 非主属性直接依赖于主键，无传递依赖

---

## 4. 物理表结构

### 4.1 住房标准表 (house_standard)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| area | DECIMAL(10,2) | PRIMARY KEY | 住房面积，唯一标识 |
| min_score | INT | NOT NULL | 最低住房分数阈值 |

```sql
CREATE TABLE house_standard (
    area DECIMAL(10,2) PRIMARY KEY COMMENT '住房面积',
    min_score INT NOT NULL COMMENT '最低住房分数阈值'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='住房标准表';
```

### 4.2 房产表 (house)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| house_no | VARCHAR(50) | PRIMARY KEY | 房号 |
| area | DECIMAL(10,2) | NOT NULL | 住房面积 |
| status | INT | NOT NULL | 0-空闲, 1-已分配 |
| rent_per_sqm | DECIMAL(10,2) | NOT NULL | 每平米房租 |

```sql
CREATE TABLE house (
    house_no VARCHAR(50) PRIMARY KEY COMMENT '房号',
    area DECIMAL(10,2) NOT NULL COMMENT '住房面积',
    status INT NOT NULL COMMENT '0-空闲 1-已分配',
    rent_per_sqm DECIMAL(10,2) NOT NULL COMMENT '每平米房租'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='房产表';
```

### 4.3 住户表 (resident)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| owner_name | VARCHAR(50) | PRIMARY KEY | 户主姓名 |
| department | VARCHAR(100) | NOT NULL | 部门 |
| title | VARCHAR(50) | NOT NULL | 职称 |
| family_size | INT | NOT NULL | 家庭人口 |
| score | INT | NOT NULL | 住房分数 |
| house_no | VARCHAR(50) | FK->house(house_no) | 房号 |
| house_area | DECIMAL(10,2) | NOT NULL | 住房面积 |

```sql
CREATE TABLE resident (
    owner_name VARCHAR(50) PRIMARY KEY COMMENT '户主姓名',
    department VARCHAR(100) NOT NULL COMMENT '部门',
    title VARCHAR(50) NOT NULL COMMENT '职称',
    family_size INT NOT NULL COMMENT '家庭人口',
    score INT NOT NULL COMMENT '住房分数',
    house_no VARCHAR(50) NOT NULL COMMENT '房号',
    house_area DECIMAL(10,2) NOT NULL COMMENT '住房面积',
    FOREIGN KEY (house_no) REFERENCES house(house_no)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='住户表';
```

### 4.4 房租表 (rent_bill)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | AUTO_INCREMENT PK | 账单ID |
| house_no | VARCHAR(50) | FK->house(house_no) | 房号 |
| owner_name | VARCHAR(50) | FK->resident(owner_name) | 户主 |
| bill_month | VARCHAR(10) | NOT NULL | 账单月份 |
| rent_amount | DECIMAL(10,2) | NOT NULL | 房租金额 |
| create_time | DATE | 可空 | 创建时间 |

```sql
CREATE TABLE rent_bill (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '账单ID',
    house_no VARCHAR(50) NOT NULL COMMENT '房号',
    owner_name VARCHAR(50) NOT NULL COMMENT '户主姓名',
    bill_month VARCHAR(10) NOT NULL COMMENT '账单月份',
    rent_amount DECIMAL(10,2) NOT NULL COMMENT '房租金额',
    create_time DATE COMMENT '创建时间',
    FOREIGN KEY (house_no) REFERENCES house(house_no),
    FOREIGN KEY (owner_name) REFERENCES resident(owner_name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='房租表';
```

### 4.5 申请表 (application)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| apply_no | VARCHAR(50) | PRIMARY KEY | 申请单号 |
| applicant_name | VARCHAR(50) | NOT NULL | 申请人 |
| department | VARCHAR(100) | NOT NULL | 部门 |
| apply_type | INT | NOT NULL | 1-分房 2-调房 3-退房 |
| apply_status | INT | NOT NULL | 0-待处理 1-已通过 2-已驳回 |
| req_area | DECIMAL(10,2) | 可空 | 要求面积 |
| old_house_no | VARCHAR(50) | 可空 | 原房号 |
| old_house_area | DECIMAL(10,2) | 可空 | 原面积 |
| title | VARCHAR(50) | 可空 | 职称 |
| family_size | INT | 可空 | 家庭人口 |
| score | INT | 可空 | 住房分数 |
| reject_reason | VARCHAR(500) | 可空 | 驳回原因 |
| create_time | DATETIME | 可空 | 创建时间 |
| process_time | DATETIME | 可空 | 处理时间 |

```sql
CREATE TABLE application (
    apply_no VARCHAR(50) PRIMARY KEY COMMENT '申请单号',
    applicant_name VARCHAR(50) NOT NULL COMMENT '申请人姓名',
    department VARCHAR(100) NOT NULL COMMENT '部门',
    apply_type INT NOT NULL COMMENT '1-分房 2-调房 3-退房',
    apply_status INT NOT NULL COMMENT '0-待处理 1-已通过 2-已驳回',
    req_area DECIMAL(10,2) COMMENT '要求面积',
    old_house_no VARCHAR(50) COMMENT '原房号',
    old_house_area DECIMAL(10,2) COMMENT '原面积',
    title VARCHAR(50) COMMENT '职称',
    family_size INT COMMENT '家庭人口',
    score INT COMMENT '住房分数',
    reject_reason VARCHAR(500) COMMENT '驳回原因',
    create_time DATETIME COMMENT '创建时间',
    process_time DATETIME COMMENT '处理时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='申请表';
```

## 6. 约束与完整性

| 约束类型 | 说明 | 实现方式 |
|---------|------|---------|
| 实体完整性 | 每表均有 PRIMARY KEY | DDL |
| 参照完整性 | resident/rent_bill 外键引用 house | FOREIGN KEY |
| 业务约束 | apply_type in (1,2,3) | 程序校验 |
| 业务约束 | status in (0,1) | 程序校验 |
| 业务约束 | score >= 对应面积阈值 | 提交时校验 |

## 7. 索引设计

| 表 | 索引字段 | 说明 |
|------|---------|------|
| house | status | 加速空房查询 |
| house | area | 加速面积匹配 |
| rent_bill | owner_name | 加速按住户查账单 |
| rent_bill | bill_month | 加速按月查账单 |
| application | (apply_type, apply_status) | 复合索引，加速队列查询 |

## 9. 数据库文件说明

| 文件 | 位置 | 用途 |
|------|------|------|
| 建表.sql | 数据库/ | 手动建库（MySQL命令行/Workbench执行） |
| 初始化数据.sql | 数据库/ | 手动插入测试数据 |
| schema.sql | resources/ | **Spring Boot 自动**建表（启动时执行） |
| data.sql | resources/ | **Spring Boot 自动**插入数据（启动时执行） |

**注意**：`schema.sql` 和 `data.sql` 必须放在 `src/main/resources/` 目录下，Spring Boot 启动时自动检测并执行。如果不想用自动初始化，可以随时删除或改名这两个文件，改用 `数据库/建表.sql` 和 `数据库/初始化数据.sql` 手动操作。
